# M4 (Delivery + Tracking)

## Scope
Driver Assignment, Real-time Tracking, Delivery Lifecycle, Reviews

## Actors
Driver (availability, accept/reject, location, status), Customer (track, review)

## APIs
POST /driver/online
POST /driver/offline
POST /driver/deliveries/{id}/accept
POST /driver/deliveries/{id}/reject
PATCH /driver/deliveries/{id}/status
POST /driver/location

GET /tracking/{orderUuid}

POST /orders/{uuid}/review

GET  /admin/deliveries
POST /admin/deliveries/{id}/reassign

## Entities
Delivery(order_id, driver_id, status, eta_minutes, distance_miles, assignment_attempts)
DriverLocationLog(driver_id, delivery_id, lat, lng, recorded_at)
ReviewAndRating(order_id, customer_id, restaurant_id, driver_id, restaurant_rating, driver_rating)

## Rules
- Delivery record created when order reaches ready_for_pickup
- Only drivers with availability=online and approval_status=approved receive requests
- Assignment considers proximity and availability
- Driver reject or no-response → reassign to next driver
- Max reassignment attempts from platform_config.max_driver_assignment_attempts (default 3) — then escalate to admin
- Distance stored in miles (decimal 6,2) — UK standard, never km
- Driver pushes location every platform_config.driver_location_update_interval_seconds (default 5s)
- Tracking response includes driver live position, order status, and ETA
- If GPS unavailable — fall back to last known location, degrade gracefully
- Review only allowed after order status = delivered; one review per order
- restaurants.rating and total_reviews updated on each new review

## Delivery Status
pending_assignment → assigned → picked_up → delivered / failed

## Output
Driver:
- Go online/offline
- Receive, accept, or reject delivery requests
- Mark picked up and delivered
- Push GPS coordinates every 5 seconds

Customer:
- Real-time driver location and order status tracking
- ETA displayed
- Submit restaurant and driver rating after delivery

Admin:
- View all deliveries with assignment attempt counts
- Manually reassign a delivery
- Escalation alert when max attempts reached
