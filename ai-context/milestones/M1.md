# M1 (Foundation)

## Scope
Auth, Profiles, RBAC, Base System

## Actors
Customer, Driver, Restaurant, Admin

## APIs
POST /auth/otp/request
POST /auth/otp/verify
POST /auth/login
POST /auth/logout

GET /profile
PUT /profile

GET  /admin/users
PATCH /admin/users/{id}/status
GET  /admin/documents
PATCH /admin/documents/{id}/approve
PATCH /admin/documents/{id}/reject
GET  /admin/config
PATCH /admin/config/{key}

## Entities
User(id, mobile, email, password, status)
CustomerProfile(user_id, name, addresses)
DriverProfile(user_id, vehicle, availability, approval_status)
Restaurant(user_id, business_details, status, approval_status)
Document(documentable, type, verification_status, verified_by)
OtpCode(mobile, code_hash, expires_at, used_at)
PlatformConfig(key, value)
AuditLog(user_id, action, entity, old_values, new_values)

## Rules
- Customer and Driver login via mobile OTP only — no password
- Restaurant and Admin login via email + password credentials
- OTP stored as bcrypt hash (code_hash) — never plaintext; verify with Hash::check()
- Role managed via Spatie laravel-permission — no role column on users
- Roles: customer, driver, restaurant_owner, admin
- Driver and Restaurant accounts start as pending_approval until admin approves
- Each document verified individually by admin — account active only when all required docs approved
- All admin mutations logged to audit_logs

## Output
- All users can register and log in via appropriate auth method
- Profiles created on registration
- Admin can approve/reject individual documents
- Admin can suspend or activate any user
- Platform config seeded and manageable
