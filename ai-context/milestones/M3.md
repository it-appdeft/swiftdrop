# M3 (Payment + Admin)

## Scope
Payments, Offers, Admin Dashboard, Reports

## Actors
Customer (pay, apply offers), Admin (manage everything)

## APIs
POST /payments/initiate
POST /payments/verify
POST /payments/{uuid}/refund

POST /offers/validate
POST /cart/apply-offer
DELETE /cart/offer

GET  /admin/orders
PATCH /admin/orders/{uuid}/status
POST /admin/orders/{uuid}/cancel

GET  /admin/users
PATCH /admin/users/{id}/status

GET    /admin/offers
POST   /admin/offers
PUT    /admin/offers/{id}
DELETE /admin/offers/{id}

GET   /admin/config
PATCH /admin/config/{key}

GET /admin/reports/orders
GET /admin/reports/restaurants
GET /admin/reports/drivers

## Entities
Payment(uuid, order_id, method, status, amount, currency, gateway_txn_id)
Offer(code, type, value, min_order_value, applicable_to, max_uses, valid_from, valid_until)
OfferUsage(offer_id, user_id, order_id, discount_applied)

## Rules
- Payment methods: card, open_banking, cod, wallet (open_banking is UK-specific)
- Card/open_banking: payment must succeed before order is created — no order on failed payment
- COD: order created immediately, payment status updated on delivery
- Platform never stores raw card data — gateway tokenises; store only gateway_txn_id
- Payment UUID is generated client-side for idempotency — duplicate requests return existing record
- Offer validation: active, within date range, min_order_value met, applicable entity matched, usage limits not exceeded
- Only one offer per order
- Refund before preparing → full refund; after preparing → partial/none per platform policy
- Restaurant rejection after payment → automatic full refund
- VAT must appear on receipts (UK legal requirement)
- All admin mutations logged to audit_logs with old/new values

## Output
- Payment initiation and gateway webhook working
- COD flow working without gateway
- Duplicate webhook calls handled idempotently
- Offer validation and application working with all eligibility checks
- Admin can view and override any order, manage users, manage offers, update config
- Basic reports accessible to admin only
- All admin actions written to audit_logs
