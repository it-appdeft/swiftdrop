# M2 (Discovery + Order)

## Scope
Discovery, Menu, Cart, Order

## Actors
Customer (browse, cart, order), Restaurant (accept, status updates)

## APIs
GET /restaurants
GET /restaurants/{id}

GET /restaurants/{id}/menu
GET /menu/items/{id}

GET  /cart
POST /cart/items
PUT  /cart/items/{id}
DELETE /cart/items/{id}
DELETE /cart

POST /orders
GET  /orders
GET  /orders/{uuid}
POST /orders/{uuid}/cancel

GET  /restaurant/orders
PATCH /restaurant/orders/{uuid}/accept
PATCH /restaurant/orders/{uuid}/reject
PATCH /restaurant/orders/{uuid}/status

POST /restaurant/menu/categories
POST /restaurant/menu/items
PUT  /restaurant/menu/items/{id}
PATCH /restaurant/menu/items/{id}/availability

## Entities
Restaurant(id, name, location, status)
MenuCategory(restaurant_id, name, sort_order)
MenuItem(restaurant_id, category_id, price, is_available, allergens)
MenuItemImage(menu_item_id, file_path, sort_order)
Cart(user_id, restaurant_id)
CartItem(cart_id, menu_item_id, quantity, unit_price)
Order(uuid, user_id, restaurant_id, status, totals, delivery_address)
OrderItem(order_id, menu_item_id, name, unit_price, quantity)

## Rules
- One cart per user; cart locked to one restaurant at a time
- Menu items can have multiple images — sort_order 0 is the primary image
- allergens on menu items follow UK law (14 mandatory allergens)
- unit_price and name on order_items are snapshots at order time — not live references
- delivery_address on orders is a JSON snapshot — not a FK to customer_addresses
- VAT calculated from platform_config.vat_rate and stored on the order
- Order UUID is the public key — never expose integer id to clients
- Validate item availability and restaurant active status before creating order
- Customer can only cancel before status reaches preparing
- Restaurant has a response timeout (platform_config.restaurant_accept_timeout_minutes) — auto-cancel on expiry

## Order Status (M2 scope)
placed → accepted → preparing → ready_for_pickup

## Output
Customer:
- Browse and search restaurants
- View menu with allergens and images
- Add to cart with restaurant lock enforced
- Place order with VAT and delivery fee calculated
- Track order status by UUID
- Cancel before preparation

Restaurant:
- Accept or reject incoming orders
- Advance status: Accepted → Preparing → Ready for Pickup
- Manage menu items, categories, images, availability
